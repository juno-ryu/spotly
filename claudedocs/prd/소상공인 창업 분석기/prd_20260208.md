## 1. Executive Summary

소상공인 창업 분석기는 공공데이터 API와 MCP 서버를 활용하여 창업 예정자에게 데이터 기반의 창업 의사결정을 지원하는 B2C/B2B 플랫폼이다.

"이 동네에서 치킨집을 열면 성공할까?"라는 질문에 객관적 데이터로 답한다. 국민연금공단 사업장 가입 내역(NPS), 국세청 사업자등록정보(NTS), 부동산 실거래가, 통계청 KOSIS 등의 공공데이터를 AI로 분석하여 상권 분석 리포트를 제공한다.

| 타겟 시장 | 초기 비용 | MVP 기간 | 예상 BEP |
|-----------|----------|---------|---------|
| 연 100만 창업자 | 낮음 (공공API 무료) | 8~12주 | 6개월 |

## 2. Problem Statement

### 2.1 현황 및 문제점

한국에서는 매년 약 100만 명이 창업하지만, 3년 내 폐업률이 80%에 달한다. 핵심 원인은 데이터 없이 감에 의존한 창업 의사결정이다.

- 상권 분석 없이 감으로 업종/입지 선택
- 경쟁 업체 현황을 수작업으로 파악 (발품 팔고 돌아다니며 세기)
- 기존 상권분석 서비스는 유동인구 중심으로 실제 영업 상태를 반영하지 못함
- 프랜차이즈 본사의 출점 분석도 제한적인 데이터에 의존

### 2.2 기존 솔루션의 한계

| 서비스 | 제공 데이터 | 한계 |
|--------|-----------|------|
| **서울시 골목상권 분석** (golmok.seoul.go.kr) | 45종 빅데이터: 점포수, 개폐업률, 추정매출, 유동인구, 상주/직장인구, 임대시세, 상권변화지표(4단계), 상권발달지수(5개), GIS 시각화. **무료** | **서울 한정** (전국 불가), 사업장별 직원 수 변동 미제공, 실시간 사업자 상태 검증 불가, AI 종합 판단 없음, 데이터 갱신 주기 분기/반기 단위 |
| 소상공인시장진흥공단 | 상권분석 보고서 | 업데이트 주기 느림, 실시간 변화 반영 불가 |
| 카카오맵 상권분석 | 유동인구, 매출 추정 | 업종별 실제 경영 상태(직원 수, 폐업률) 미반영 |
| 인구소멸 지역 분석 | 인구 통계 | 상권 활력도, 경쟁 업체 건강성 분석 없음 |

> **핵심 경쟁자: 서울시 골목상권 분석 서비스**
>
> golmok.seoul.go.kr은 가장 강력한 기존 솔루션이다. 100개 생활밀접업종에 대해 점포수, 개폐업률, 추정매출, 유동/상주/직장인구, 임대시세, 상권변화지표까지 무료 제공한다. 단, 서울 한정이며 "사업장이 실제로 건강한가"(직원 수 변동, 실시간 영업상태)라는 핵심 질문에는 답하지 못한다. 우리 서비스는 이 서비스를 데이터 소스로 활용하면서, 동시에 golmok이 제공하지 못하는 영역에 집중한다.

### 2.3 우리의 차별점 (vs. golmok.seoul.go.kr)

| 비교 항목 | 서울시 골목상권 분석 | 소상공인 창업 분석기 (본 서비스) |
|----------|-------------------|-------------------------------|
| 지역 범위 | 서울시만 | **전국** |
| 사업장 직원 수 추적 | 불가 | **NPS MCP로 월별 추적** (상권 활력도 선행지표) |
| 실시간 사업자 상태 | 분기/반기 개폐업 통계 | **NTS MCP로 실시간 영업/휴업/폐업 확인** |
| 추정 평균 급여 | 불가 | **NPS MCP로 사업장별 추정 급여** |
| AI 종합 판단 | 없음 (사용자가 직접 해석) | **AI가 "창업 추천/비추천" 판단 + 리포트** |
| 트렌드 알림 | 없음 | **관심 지역 변화 실시간 Push 알림** |
| B2B API | 별도 Open API 존재 | **가공된 분석 데이터 API 판매** |
| 가격 | 무료 | 기본 무료 + 프리미엄 유료 |

핵심 차별점: NPS MCP로 "같은 동네 같은 업종 사업장이 최근 직원을 늘리고 있는지, 줄이고 있는지"를 월별로 추적 → golmok의 분기 단위 개폐업 통계보다 훨씬 빠르고 세밀한 상권 건강도 파악이 가능하다.

## 4. Scope & Features

### 4.1 데이터 소스 및 API/MCP 조합

| 구분 | 데이터 소스 | 활용 목적 | 연동 방식 |
|------|-----------|----------|----------|
| **MCP** | NPS Business Enrollment | 지역/업종별 사업장 수, 직원 수, 평균 급여 추정 | MCP Server (Python) |
| **MCP** | NTS Business Verification | 동일 업종 폐업률 분석, 영업 상태 확인 | MCP Server (Python) |
| API | 아파트 매매/전월세 실거래가 | 상권 주변 주거 밀도, 소득 수준 추정 | REST API (data.go.kr) |
| API | 통계청 KOSIS | 지역별 인구, 소득, 소비 패턴 | REST API |
| **데이터** | **서울시 골목상권 분석 (golmok)** | **추정매출, 개폐업률, 유동인구, 상권변화지표, 상권발달지수 (서울 한정)** | **Open API (data.seoul.go.kr)** |
| API | 서울열린데이터/경기데이터드림 | 유동인구, 상권 분석 | REST API |
| API | 네이버/카카오 지도 | 반경 내 경쟁업체 시각화, 지도 UI | REST API |

### 4.2 핵심 기능 (MVP)

#### F1. 상권 건강도 분석

- 특정 위치 + 업종 입력 시 주변 동일 업종 사업장 현황 조회 (NPS MCP)
- 직원 수 증감 추이를 월별로 추적하여 상권 활력도 스코어링
- 주변 사업장 폐업률 분석 (NTS MCP)

#### F2. 경쟁 강도 분석

- 반경 500m/1km/3km 내 동일 업종 사업장 수 및 지도 시각화
- 경쟁 업체 직원 수 비교로 규모 파악
- 최근 1년 신규 개업/폐업 비율 분석

#### F3. 입지 적합성 스코어

- 주변 주거 밀도 및 소득 수준 (실거래가 기반)
- 지역 인구 추이 (KOSIS)
- 유동인구 및 시간대별 분포
- 서울 지역의 경우 golmok 상권발달지수(매출/인프라/가맹점/인구/금융 5개 지수) 연동
- 종합 입지 적합성 점수 (100점 만점) 산출

#### F4. AI 분석 리포트 생성

- 수집된 데이터를 종합하여 "창업 추천/비추천" AI 판단
- 경쟁 현황, 위험 요소, 기회 요소 요약 리포트
- PDF/웹 형식 보고서 다운로드

### 4.3 확장 기능 (Phase 2~3)

| Phase | 기능 | 설명 |
|-------|------|------|
| Phase 2 | 트렌드 알림 서비스 | 관심 지역/업종의 경쟁업체 변화 실시간 알림 (Push) |
| Phase 2 | 예상 매출 시뮬레이션 | 입지 조건 기반 예상 월 매출/소득 시뮬레이션 |
| Phase 3 | 프랜차이즈 본사 대시보드 | 전국 가맹점 모니터링 + 신규 출점 자동 추천 |
| Phase 3 | B2B API 제공 | 은행/보증기관에 상권 분석 데이터 API 판매 |

## 5. User Flow

1. 사용자가 웹/앱에 접속하여 회원가입 또는 비회원 체험 선택
2. 업종 선택 (예: 치킨/커피/미용실 등 카테고리)
3. 위치 입력 (주소 검색 또는 지도에서 핀 드롭)
4. 분석 반경 설정 (500m / 1km / 3km)
5. AI 분석 실행 → 로딩 화면 (MCP 서버 + API 동시 호출)
6. 결과 대시보드 확인 (지도 + 차트 + 스코어)
7. 상세 분석 리포트 다운로드 (프리미엄 기능)

## 7. Technical Architecture

### 7.1 시스템 구성도

시스템은 크게 4개 레이어로 구성된다.

| 레이어 | 구성 요소 | 기술 스택 |
|--------|---------|----------|
| Frontend | 웹 대시보드, 지도 UI, 분석 결과 시각화 | Next.js + TypeScript + Tailwind CSS |
| Backend API | REST API 서버, 인증, 결제 연동 | FastAPI (Python) / Node.js |
| Data Layer | MCP 서버 연동, 공공API 호출, 데이터 캐싱 | data-go-mcp-servers + Redis + PostgreSQL |
| AI/Analytics | AI 분석 엔진, 리포트 생성 | Claude API / CLOVA Studio |

### 7.2 MCP 서버 활용 아키텍처

data-go-mcp-servers 패키지를 백엔드에 통합하여 사용한다.

- **NPS MCP**: 사업장 검색 (`search_business`) → 상세정보 (`get_business_detail`) → 기간별 현황 (`get_period_status`)
- **NTS MCP**: 사업자 상태 조회 (`check_business_status`) → 대량 검증 (`batch_validate_businesses`)
- 두 MCP를 병렬 호출하여 응답 속도 최적화 (목표: 3초 이내 결과 반환)

### 7.3 데이터 파이프라인

```
1. 사용자 입력 (Address + Business Type)
   ↓
2. Geocoding → 좌표 변환 → 서울 여부 판별
   ↓
3. NPS MCP → 반경 내 동일 업종 사업장 목록 조회
   │
4. NTS MCP → 각 사업장 영업상태 확인 (활성/휴업/폐업)  ← 병렬 호출
   │
5. 실거래가 API → 주변 부동산 시세 및 소득 수준 추정     ← 병렬 호출
   │
6. KOSIS API → 지역 인구 통계 조회                      ← 병렬 호출
   │
7. [서울] golmok Open API → 추정매출, 개폐업률,          ← 병렬 호출
   │  상권변화지표, 상권발달지수
   ↓
8. 지도 API → 경쟁업체 위치 시각화
   ↓
9. AI 엔진 → 종합 분석 및 리포트 생성
```

## 8. 창업 적합성 스코어링 모델

종합 점수는 100점 만점으로, 다음 5개 지표를 가중 합산하여 산출한다.

| 지표 | 가중치 | 데이터 소스 | 산출 방법 |
|------|--------|-----------|----------|
| 상권 활력도 | 30% | NPS MCP (`get_period_status`) | 주변 사업장 직원 수 증감률 기반 |
| 경쟁 강도 | 25% | NPS MCP (`search_business`) | 반경 내 동일 업종 밀도 (적을수록 유리) |
| 생존율 | 20% | NTS MCP (`check_business_status`) | 폐업률 낮을수록 높은 점수 |
| 주거 밀도 | 10% | 실거래가 API + KOSIS | 주변 세대 수 및 인구 변동 |
| 소득 수준 | 5% | 실거래가 API | 주변 아파트 시세 기반 추정 소득 |
| 상권발달지수 | 10% | golmok Open API (서울) / 대체지표 (비서울) | 매출·인프라·인구·금융 종합지수 (서울 한정, 비서울은 KOSIS 대체) |

**점수 구간**:
- 80~100점: 강력 추천
- 60~79점: 조건부 추천
- 40~59점: 주의 필요
- 0~39점: 비추천

| # | 위험 요소 | 영향도 | 완화 방안 |

| R1 | 공공API 응답 속도 지연 | 🔴 높음 | Redis 캐싱 도입 (TTL 24시간), MCP 병렬 호출로 대기 시간 최소화 |
| R2 | API 호출 제한 (Rate Limit) | 🟡 중간 | NTS MCP `batch_validate` 활용 (100건 배치), 요청 큐 관리 |
| R3 | API 정책 변경/중단 | 🟡 중간 | 데이터 수집층 추상화로 대체 API 전환 용이, 원본 데이터 주기적 백업 |
| R4 | golmok 등 무료 서비스와의 경쟁 | 🟡 중간 | golmok 데이터를 소스로 활용하며 공존. NPS/NTS 기반 "사업장 건강도"와 AI 판단이라는 상위 레이어로 차별화. 전국 커버리지가 핵심 해자 |
| R5 | 데이터 정확도 이슈 | 🟡 중간 | AI 분석 결과에 신뢰도 점수 표시, 여러 데이터 소스 교차 검증 |

## 12. Non-Functional Requirements

- **응답 시간**: 분석 결과 3초 이내 반환 (캐시 적중 시), 최초 분석 10초 이내
- **가용성**: 월간 SLA 99.5% 이상
- **보안**: 개인정보 미수집, 사업자등록번호 암호화 저장, HTTPS 필수
- **확장성**: 동시 접속자 1,000명 이상 처리 가능한 아키텍처
- **접근성**: 모바일 반응형 웹, 웹 접근성 지침(WCAG 2.1 AA) 준수
- **데이터 갱신**: NPS/NTS 데이터 최소 일 1회 갱신, 부동산/통계 데이터 월 1회 갱신
