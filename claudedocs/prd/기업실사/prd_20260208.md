## 1. Executive Summary

기업 실사 AI 플랫폼은 사업자번호 하나만 입력하면 **"이 회사 거래해도 될까?"를 3분 안에 판단**해주는 B2B SaaS 서비스다.

국세청(NTS), 국민연금공단(NPS), 금융위원회(FSC) 공공데이터 MCP 서버 3종을 조합하여 사업자 진위확인 → 재무제표 분석 → 직원 수 추이 분석을 자동화하고, AI가 종합 리스크 스코어와 실사 리포트를 생성한다.

기존 기업 실사는 수작업으로 3~7일이 걸리고, 건당 수백만 원의 컨설팅 비용이 발생한다. 본 서비스는 이를 **3분, 건당 3,000~10,000원**으로 압축한다.

| 항목 | 내용 |
|------|------|
| 타겟 시장 | 기업 신용정보 시장 (국내 약 3조원+) |
| 핵심 데이터 | NTS + NPS + FSC MCP 서버 3종 조합 |
| 초기 비용 | 낮음 (공공API 무료, LLM API 비용만) |
| MVP 기간 | 10~14주 |
| 예상 BEP | 8~12개월 |

## 2. Problem Statement

### 2.1 현황 및 문제점

한국에서 기업 간 거래(B2B)를 시작할 때 상대방의 신뢰도를 파악하는 것은 필수적이지만, 현실은 다음과 같다.

**시간 문제**: 전통적 기업 실사는 3~7일 소요. VC/PE의 투자 심사는 2~4주가 걸린다. 이 기간 동안 거래 기회를 놓치거나 의사결정이 지연된다.

**비용 문제**: 전문 실사 보고서 1건에 50만~500만원. 중소기업과 스타트업이 거래처마다 이 비용을 감당하기 어렵다. 결과적으로 실사 없이 거래하다가 사기나 부도에 당한다.

**정보 파편화**: 사업자등록 상태는 국세청, 재무제표는 금융위, 직원 수는 국민연금공단, 소송 이력은 법원에 흩어져 있다. 이를 하나로 모아보려면 각각 다른 시스템에 접속해야 한다.

**판단 부재**: 데이터를 모아도 "그래서 이 회사가 안전한가?"에 대한 종합 판단은 결국 사람이 해야 한다. 그런데 대부분의 중소기업 담당자는 재무제표를 읽을 전문성이 없다.

### 2.2 시장 검증 신호

이 문제가 실재한다는 증거는 이미 시장에 있다.

- **NICE평가정보 그룹** 연 매출 3조원+ (기업 신용정보 시장의 압도적 1위)
- **캐시노트(한국신용데이터)** 매출 1,380억원, 유니콘 등극, 140만 사업장 데이터 확보
- **팝빌** 950만+ 기업 데이터, 사업자 조회 API 시장에서 독보적 성장
- **크레딧잡/원티드인사이트** 국민연금 데이터 기반 240만 기업 정보, 월 수억 매출

### 3.4 핵심 차별화 — 왜 우리가 이길 수 있는가

**1. MCP 서버 3종 콤보 = 유일한 실시간 통합 파이프라인**

기존 서비스들은 각각 하나의 데이터 소스에 묶여 있다. 우리는 NTS(사업자 상태) + NPS(고용 건강성) + FSC(재무 건강성)를 하나의 파이프라인으로 통합한 최초의 서비스다.

| 확인 항목 | NTS MCP | NPS MCP | FSC MCP | 통합 판단 |
|----------|---------|---------|---------|----------|
| 이 회사 실존하나? | ✅ 진위확인 | - | - | 1차 필터 |
| 영업 중인가? | ✅ 상태조회 | - | - | 2차 필터 |
| 직원이 늘고 있나? | - | ✅ 가입자수 추이 | - | 성장 지표 |
| 급여 수준은? | - | ✅ 추정 평균급여 | - | 안정성 지표 |
| 돈을 벌고 있나? | - | - | ✅ 손익계산서 | 수익성 지표 |
| 빚은 감당 가능한가? | - | - | ✅ 재무상태표 | 리스크 지표 |
| **종합: 거래해도 되나?** | ✅ | ✅ | ✅ | **AI 리스크 스코어** |

**2. 정부가 절대 만들 수 없는 서비스**

정부/공공기관은 "이 회사는 위험하다"는 판단을 공식적으로 내릴 수 없다. 이는 민간 기업의 영업권을 침해하고, 오판 시 국가배상 소송 대상이 된다. 따라서 정부는 원천 데이터만 공개할 수 있고, **리스크 판단은 민간 영역의 독점적 기회**다.

**3. 가격 파괴**

| 서비스 | 기업 1곳 분석 비용 | 소요 시간 |
|--------|-------------------|----------|
| 전문 컨설팅사 | 200만~500만원 | 1~2주 |
| NICE/KED 보고서 | 5,000~50,000원 | 수시간 (수동) |
| PitchBook | 연 1,200만~2,000만원 (무제한) | 수시간 (수동) |
| **본 서비스** | **3,000~10,000원** | **3분** |

## 5. Product Scope & Features

### 5.1 MVP (Phase 1) — 핵심 실사 리포트

**입력**: 사업자등록번호 (10자리)

**출력**: AI 기업 실사 리포트

```
┌─────────────────────────────────────────────┐
│  기업 실사 리포트 — (주)ABC테크              │
│  사업자번호: 123-45-67890                    │
│  생성일시: 2026.02.08 14:30                  │
├─────────────────────────────────────────────┤
│                                             │
│  📊 리스크 스코어: 72/100 (보통)             │
│  ████████████████████░░░░░░░░               │
│                                             │
│  ✅ 사업자 상태: 영업중 (확인됨)             │
│  📈 직원 추이: 23명→31명 (+34%, 12개월)      │
│  💰 추정 평균급여: 380만원 (업종 상위 30%)    │
│  📊 매출: 45억 (전년비 +22%)                 │
│  ⚠️ 부채비율: 180% (업종 평균 120%)          │
│                                             │
│  🤖 AI 종합 의견:                            │
│  "성장세가 뚜렷하나 부채비율이 업종 평균      │
│   대비 높아 주의 필요. 직원 증가와 급여        │
│   수준은 긍정적. 단기 거래는 안전하나          │
│   장기 파트너십 시 재무 개선 추이 모니터링     │
│   권장."                                     │
│                                             │
│  [상세 재무분석] [업종 비교] [PDF 다운로드]    │
└─────────────────────────────────────────────┘
```

**MVP 기능 목록**:

| 기능 | 데이터 소스 | 우선순위 |

### 5.2 Phase 2 — 모니터링 & 알림

- **워치리스트**: 관심 기업 등록 시 변동사항 자동 알림 (직원 급감, 휴업 전환, 재무 악화 등)
- **거래처 포트폴리오**: 모든 거래처를 한 대시보드에서 리스크 모니터링
- **슬랙/이메일 알림**: "거래처 (주)DEF의 직원이 3개월간 30% 감소했습니다" 자동 알림

### 5.3 Phase 3 — 심화 분석

- **나라장터 연동 (PPS MCP)**: 정부 조달 실적으로 기업 안정성 추가 검증
- **소송/판례 검색**: 대법원 API 연동으로 법적 리스크 분석
- **뉴스 감성 분석**: 기업 관련 뉴스의 긍부정 트렌드
- **업종 벤치마킹 리포트**: 동종업계 내 순위 분석
- **API 제공**: 고객사 자체 시스템에 실사 기능 내장 (팝빌 모델)

| 상황 | 처리 방식 |
|------|----------|

## 8. Technical Architecture

### 8.1 시스템 구성도

```
┌─────────────────────────────────────────────────────────┐
│                      Frontend                            │
│          Next.js (Web) / React Native (Mobile)           │
└───────────────────────┬─────────────────────────────────┘
                        │ REST API / WebSocket
┌───────────────────────┴─────────────────────────────────┐
│                    API Gateway                           │
│              (Rate Limiting, Auth, Billing)               │
└───────────────────────┬─────────────────────────────────┘
                        │
┌───────────────────────┴─────────────────────────────────┐
│                  Orchestration Layer                      │
│           (MCP 호출 순서 제어, 병렬 처리)                  │
├──────────┬──────────┬──────────┬────────────────────────┤
│ NTS MCP  │ NPS MCP  │ FSC MCP  │ 한국은행 API / 기타     │
│ 사업자확인│ 고용정보  │ 재무정보  │ 업종 평균 데이터        │
└──────────┴──────────┴──────────┴────────────────────────┘
                        │
┌───────────────────────┴─────────────────────────────────┐
│                  AI Analysis Engine                       │
│  - 리스크 스코어링 모델 (Rule-based + ML)                 │
│  - 업종 벤치마킹 엔진                                     │
│  - LLM 리포트 생성 (Claude/GPT)                          │
└───────────────────────┬─────────────────────────────────┘
                        │
┌───────────────────────┴─────────────────────────────────┐
│                    Data Layer                             │
│  PostgreSQL (기업 캐시) + Redis (세션) + S3 (리포트)       │
└─────────────────────────────────────────────────────────┘
```

### 8.2 MCP 서버 활용 상세

| MCP 서버 | 호출 목적 | 주요 엔드포인트 | 응답 시간 |
|----------|----------|----------------|----------|
| NTS Business Verification | 사업자 진위확인, 영업상태 | `validate`, `status` | <1초 |
| NPS Business Enrollment | 사업장 검색, 직원수, 급여 | `search`, `employees`, `salary` | 1~3초 |
| FSC Financial Info | 요약재무제표, 대차대조표, 손익계산서 | `summary`, `balance`, `income` | 2~5초 |

**총 데이터 수집 시간**: 약 5~10초 (병렬 호출 시)
**AI 분석 + 리포트 생성**: 약 10~20초
**전체 응답 시간**: 약 30초~1분 (목표 3분 이내, 충분한 마진)

### 8.3 기술 스택

| 영역 | 기술 | 선택 이유 |
|------|------|----------|
| Frontend | Next.js 14+ (App Router) | SSR, 빠른 초기 로딩 |
| Backend | Node.js (NestJS) 또는 Python (FastAPI) | MCP 서버 연동 용이 |
| AI/ML | Claude API + 자체 스코어링 모델 | 한국어 리포트 품질 |
| DB | PostgreSQL + Redis | 기업 데이터 캐싱, 세션 관리 |
| Infra | AWS (ECS + RDS + ElastiCache) | 확장성, 비용 효율 |
| 모니터링 | Datadog + Sentry | 실시간 장애 감지 |

## 12. Risks & Mitigation

| 리스크 | 심각도 | 발생 확률 | 완화 전략 |
|--------|--------|----------|----------|
| **MCP 서버 불안정/중단** | 높음 | 중간 | 자체 데이터 캐싱 레이어, 공공데이터포털 직접 API 폴백 |
| **공공데이터 정책 변경** | 높음 | 낮음 | 다중 데이터 소스 확보, 정부 동향 모니터링 |
| **NICE/KED의 가격 인하 대응** | 중간 | 중간 | AI 판단력으로 차별화 (데이터 양이 아닌 분석 품질 경쟁) |
| **LLM 환각(Hallucination)** | 높음 | 중간 | Rule-based 스코어링 병행, 수치는 원본 데이터 직접 표시, LLM은 해석만 담당 |
| **개인정보보호법 이슈** | 높음 | 낮음 | 공공데이터만 사용 (이미 공개된 정보), 법률 자문 확보 |
| **팝빌의 분석 기능 추가** | 중간 | 중간 | 선점 + AI 품질에서 격차 유지, 팝빌은 API 공급자 포지션 유지 가능성 높음 |
| **가격 경쟁 (크레딧잡 무료 모델)** | 중간 | 높음 | 프리미엄 분석 (재무 통합)으로 차별화, 크레딧잡은 NPS 단일 소스 한계 |

## 14. Go-to-Market 전략

### 14.1 Launch 전략

**Phase 0 — 클로즈드 베타 (4주)**
- VC/PE 심사역 10명, 중소기업 CFO 10명 대상 무료 체험
- 피드백 수집 후 스코어링 모델 개선
- "실사 3분 vs 3일" 비교 사례 확보

**Phase 1 — 오픈 베타 + 콘텐츠 마케팅**
- 프리미엄 리포트 샘플 공개 (실제 기업 분석 예시)
- "사업자번호로 보는 기업 건강검진" 블로그 시리즈
- 스타트업/VC 커뮤니티 (디스콰이어, 아웃스탠딩) 타겟 마케팅

**Phase 2 — B2B 영업**
- 프랜차이즈 본사 직접 영업 (가맹 심사 자동화 제안)
- 법무법인 파트너십 (채권 회수 전 재무 분석 도구)
- 중견기업 구매팀 타겟 (거래처 관리 솔루션)

### 14.2 차별화 메시지

> **"사업자번호 하나로, 3분 만에, 거래처 건강검진"**

기존 서비스들이 "데이터 조회"에 머무를 때, 우리는 "판단"을 제공한다. 의사가 혈액검사 수치를 나열하는 것이 아니라 "건강합니다" 또는 "정밀검사 필요합니다"라고 말해주는 것과 같다.
