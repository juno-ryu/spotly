# spotly — 프로젝트 목적

> 소상공인 예비창업자가 입지를 선택할 때 공공 데이터 기반의 신뢰할 수 있는 판단을 내릴 수 있도록 돕는다.

## 핵심 흐름

```
주소 + 업종 + 반경 입력
  ↓ (병렬 API 수집)
  ├─ Kakao Places    → 경쟁업체 수집
  ├─ KOSIS           → 배후 인구 (단일 지표, 로그 정규화)
  └─ 서울 골목상권   → 매출/유동인구 (서울만)
  ↓
3대 지표 점수화 (0~100) + 등급 (A~F)
  ↓
인사이트 메시지 생성 (규칙 기반)
  ↓ (선택)
AI 리포트 (Claude Haiku — 창업 컨설턴트 페르소나)
```

---

## 스코어링 엔진 현황

### 지표 구성

| 지표 | 범위 | 대상 지역 | 상태 |
|------|------|----------|------|
| 경쟁 강도 | 0~100 | 전국 | ✅ 운영 중 |
| 상권 활력도 | 0~100 | 서울만 | ✅ 운영 중 |
| 배후 인구 | 0~100 | 전국 | ✅ 운영 중 |

> ⚠️ 5대 지표를 목표로 했으나 현재 3개만 구현됨. 가중치 합산 체계 미완성.

---

### 1. 경쟁 강도 (Competition)

**데이터**: Kakao Places API (반경 내 동종업체)

**계산 방식**:
- 밀집도 점수 (90%): 매장 간 평균 거리를 업종별 기준값과 비교, 비선형 커브(^2.0) 적용
- 프랜차이즈 U커브 (10%): 비율 20~40% 최적 (상권 검증 + 개인매장 공존)

**업종별 기준 밀집도** (일부):
```
한식:    50m  커피:  150m  편의점: 110m
미용실:  90m  부동산: 90m  감자탕: 630m
```

**프랜차이즈 감지**: 하드코딩 브랜드 목록 ~150개 (fuzzy match)

---

### 2. 상권 활력도 (Vitality) — 서울 전용

**데이터**: 서울 골목상권 API (5개 테이블 병렬 수집)

**구성**:
- 점포당 매출 점수 (35%): 분기 매출 → [500만~1.5억원] 선형 정규화
- 상권변화 점수 (30%): LL(90점) / LH(70점) / HL(40점) / HH(20점)
- 유동인구 점수 (35%): 분기 유동인구 → [5만~200만명] 선형 정규화

> 유동인구 데이터 없을 시: 매출 55% + 변화 45%로 자동 전환

---

### 3. 배후 인구 (Population)

**데이터**: KOSIS 인구 통계

**구성**: 인구 단일 지표 (세대수는 읍면동/시군구 공간 해상도 불일치로 제거)

**정규화 기준**: 로그 변환 정규화 (인구 분포 우측 편향 보정)
```
읍면동: 3,000~40,000명  (중앙값 ~10,469명 → 약 48점)
시군구: 50,000~600,000명 (중앙값 ~186,882명 → 약 54점)
```

---

### 등급 체계

```
A: 80점+   B: 60~79   C: 40~59   D: 20~39   F: ~19
```

## 인사이트 시스템 현황

규칙 기반으로 생성. 현재 2개 도메인만 존재:
- `competition.ts`: 밀집도 등급 해석 + 프랜차이즈 현황
- `population.ts`: 매출 규모 + 유동인구 + 배후세대 + 폐업률

> ⚠️ 인사이트가 코드 곳곳에 흩어져 있고 통합 체계 없음. `ai-report-specialist` 담당.

---

## AI 리포트 현황

**모델**: Claude Haiku

**페르소나**: "가게를 3번 창업해본 10년 차 상권 컨설턴트"

**데이터 전달 원칙**: 점수가 아닌 원본 데이터(밀집도m, 매출액원, 인구명)를 전달 → Claude가 독립적으로 판단

**출력 스키마**:
```
verdict: 추천 / 조건부 추천 / 주의 / 비추천
summary, strengths[], risks[], recommendations[], detailedAnalysis
```

## 미구현 / 미연동 상태

| 항목 | 상태 | 비고 |
|------|------|------|
| NPS (국민연금) | ❌ 미사용 | 코드 존재하나 파이프라인 미연결 |
| 부동산 실거래 | ❌ 미사용 | XML 파서 존재하나 미연결 |
| 프랜차이즈 공정위 API | ❌ 미사용 | fallback 기본값으로 대체 중 |
| 서울 외 지역 활력도 | ❌ 없음 | 서울 골목상권 API 한계 |
| 5대 지표 완성 | ⚠️ 3/5 | 생존율, 소득수준 미구현 |
| 가중치 합산 체계 | ⚠️ 미완성 | 지표별 독립 점수만 존재 |
