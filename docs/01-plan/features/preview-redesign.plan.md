# Plan: preview-redesign (인사이트 미리보기 리디자인)

> 작성일: 2026-03-15
> 레벨: Dynamic
> 상태: Plan 작성 중

---

## 1. 목표

현재 인사이트 화면(7개 섹션 풀 노출)을 **미리보기(Preview)** 형태로 축소하고,
상세 인사이트는 AI 리포트와 통합하여 **결제 후 제공**하는 구조로 전환한다.

### 핵심 목적
- 무료 → 유료 전환율(Conversion) 극대화
- 인사이트 개발·유지 부담 감소
- 결제 플로우 삽입을 위한 자연스러운 구조 확보

---

## 2. 현황 분석

### 현재 플로우

```
웰컴 → 로그인 → 업종선택 → 지역선택 → 인사이트(전체) → AI 리포트
```

### 현재 인사이트 화면 구성

| 섹션 | 내용 | 무료 제공 |
|------|------|---------|
| 총점 + 등급 | 4대 지표 가중합산 (0~100점, A~F) | ✅ |
| 경쟁 강도 | 직접/인접 경쟁업체, 프랜차이즈 비율, 밀집도 | ✅ |
| 상권 활력도 | 매출, 상권변화지표, 유동인구 (서울) | ✅ |
| 배후 인구 | 읍면동/시군구 인구 | ✅ |
| 교통 | 지하철 승하차, 버스 노선 | ✅ |
| 학교 | 초중고 목록, 거리 | ✅ |
| 대학교 | 대학교 목록, 방학 리스크 | ✅ |
| 병의원 | 종합병원 목록 | ✅ |
| 위험 패턴 | 유령상권, 레드오션, 거품, 수요부족, 교통사각 | ✅ |

**문제**: 전부 무료 → AI 리포트 결제 동기 약화

### 기술 스택 현황

```
Next.js 16 (App Router) + TypeScript + Prisma 6 + Supabase Auth
shadcn-ui + Tailwind 4 + Recharts + Kakao Maps
Claude API (haiku-4-5) — AI 리포트 생성
```

## 3. 목표 플로우

```
웰컴 → 로그인 → 업종선택 → 지역선택 → 미리보기(Preview) → 결제 → AI 리포트(상세)
```

### 3-1. 미리보기 화면 (무료)

사용자에게 "이 서비스가 진짜 내 위치 데이터를 갖고 있다"는 신뢰를 주면서,
"왜 이 점수인지" 궁금증을 유발하는 최소한의 정보만 제공.

**포함할 내용**:
- 총점 + 등급 (예: "67점 B등급") — 가장 강력한 호기심 유발
- 핵심 지표 3개 점수 (경쟁/인구/교통 또는 서울은 활력도 포함)
- 팩트 숫자 3줄 (경쟁업체 N곳 / 유동인구 일 N만명 / 가장 가까운 역 Nm)
- CTA: "AI 전문가 분석 리포트 받기"

**제외할 내용** (결제 뒤로 이동):
- 각 섹션 상세 인사이트 (학교/대학교/병의원/버스 상세)
- 업종별 맥락 텍스트
- 위험 패턴 분석
- 등급별 해석

### 3-2. AI 리포트 (유료)

기존 인사이트 상세 + AI 종합 분석이 합쳐진 하나의 결과물.

**포함할 내용**:
- 모든 섹션 상세 인사이트 (현재 인사이트 화면 내용 그대로)
- 위험 패턴 분석
- AI 종합 리포트 (Claude 생성)
  - 기회 요인 / 위험 요인 정리
  - 업종 특화 조언
  - 실행 제안

## 4. 구현 범위

### Phase 1 — 미리보기 화면 (핵심)

| 작업 | 상세 | 담당 |
|------|------|------|
| 미리보기 컴포넌트 | `analysis-preview.tsx` — 총점+등급+핵심수치 카드 | frontend |
| 기존 인사이트 분리 | `analysis-result.tsx`를 미리보기/상세 두 모드로 분리 | frontend |
| 라우팅 변경 | `/analysis/[id]` → 미리보기, `/analysis/[id]/report` → 상세 | frontend |
| CTA 버튼 | "AI 분석 리포트 받기" 버튼 + 가격 표시 | frontend |

### Phase 2 — 결제 연동 (후속)

| 작업 | 상세 | 담당 |
|------|------|------|
| 결제 수단 연동 | 토스페이먼츠 / 카카오페이 등 PG 연동 | backend |
| 결제 모델 | Payment 테이블, 결제 상태 관리 | backend |
| 접근 제어 | 미결제 시 상세 리포트 접근 차단 | backend |
| 결제 완료 후 | AI 리포트 생성 트리거 | backend |

### Phase 3 — AI 리포트 통합 (후속)

| 작업 | 상세 | 담당 |
|------|------|------|
| 리포트 화면 통합 | 인사이트 상세 + AI 리포트를 하나의 화면으로 | frontend |
| Claude 프롬프트 개선 | 유료 품질에 맞는 프롬프트 리디자인 | ai-report |

## 5. 이번 작업 범위 (Phase 1만)

결제 연동은 아직 미개발 → **Phase 1 미리보기 화면만 먼저 구현**.
결제 버튼은 UI만 배치하고, 클릭 시 "준비 중" 또는 상세 화면으로 바로 이동 (임시).

### 변경 대상 파일 (예상)

```
src/features/analysis/components/
├── analysis-preview.tsx       ← 신규: 미리보기 컴포넌트
├── analysis-result.tsx        ← 수정: 상세 모드로 전환
└── score-summary-card.tsx     ← 신규: 총점+등급+핵심수치 카드

src/app/(main)/analysis/[id]/
├── page.tsx                   ← 수정: 미리보기 모드로 변경
└── report/
    └── page.tsx               ← 신규: 상세 리포트 페이지
```

## 7. 리스크 및 고려사항

| 리스크 | 대응 |
|--------|------|
| 무료 정보가 너무 적으면 이탈 | 총점+등급+팩트 숫자로 "데이터 있다"는 신뢰 확보 |
| 기존 사용자 혼란 | 미리보기→상세 전환을 자연스럽게 설계 |
| 결제 미구현 시 UX | Phase 1에서는 CTA → 상세 바로 이동 (임시) |
| AI 리포트 품질 | 유료화 시 프롬프트 품질 개선 필수 (Phase 3) |
| 서울 vs 비서울 | 비서울은 활력도/생존율 없음 → 미리보기 지표 2개만 표시 |
