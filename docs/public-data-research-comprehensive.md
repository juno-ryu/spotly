# 창업 입지 분석용 공공데이터 API 전수 조사 (2025-2026)

**작성일**: 2026년 2월
**조사 대상**: 현재 연동된 API를 제외한 추가 활용 가능 공공데이터
**주요 포털**: data.go.kr, data.seoul.go.kr, 각 부처 OpenAPI

---

## 📋 조사 범위 및 제외 사항

### 현재 연동된 API (제외함)
- Kakao Places (경쟁업체)
- 서울 골목상권 (서울시 열린데이터광장)
- KOSIS 통계청 인구
- 지하철 승하차 (서울시)
- 버스 정류소 (TAGO, 국토부)
- NPS 국민연금 (미연결 상태)
- 부동산 실거래 (미연결 상태)
- 공정위 프랜차이즈 (미연결 상태)

### 조사 범주
1. 유동인구 / 배후인구
2. 교육시설 (어린이집, 유치원, 학교, 대학)
3. 주거 / 부동산 특성
4. 상권 특성 (폐업, 개업, 위생, 전통시장)
5. 의료 / 복지시설
6. 교통 / 접근성
7. 행정 / 인허가 현황
8. 지리 정보 및 격자 데이터

## 1️⃣ 유동인구 / 배후인구 분석

### 1.1 KT 통신 빅데이터 - 수도권 생활이동 데이터 (신규 공개)

**상태**: 서울시 열린데이터광장에 2025년 12월부터 전면 공개 (무료)

| 항목 | 내용 |
|------|------|
| **제공기관** | KT 엔터프라이즈 |
| **데이터명** | 수도권 생활이동 데이터 (Metropolitan Mobility Data) |
| **공식명칭** | 모빌리티 데이터 (8가지 이동수단 자동판별) |
| **접근 경로** | 서울시 열린데이터광장 (data.seoul.go.kr) |
| **무료/유료** | 무료 공개 |
| **인증 방식** | 서울시 공공데이터포털 회원 가입 (기관별 신청 가능) |
| **공간 해상도** | 250m 격자 단위 (플랫폼 확대 시 50m, 500m, 1km 자동 조정) |
| **이동수단** | 항공, 기차, 고속버스, 지하철, 버스, 승용차, 도보 (자동 판별) |
| **지역 범위** | 서울, 경기, 인천 수도권 |

**활용 지표**:
- 특정 상권의 시간대별 유동인구 추이
- 이동수단별 유입/유출 비율 (대중교통 vs 자가용 판별)
- 주중/주말 유동패턴 분석
- 타지역에서의 유입 방향성 파악 (관광객 vs 통근/통학 구분)

**한계**:
- 수도권(서울, 경기, 인천)만 커버 (부산, 대구 등 외 지역 미제공)
- 실거래 고객이 아닌 통신 기지국 기반 유추값

**참고**:
- 기존 서울시 유동인구(KT 데이터 기반)보다 훨씬 정밀한 모빌리티 정보 제공
- K-데이터 인사이트 플랫폼에서 시각화 및 분석 도구 제공

|------|------|

| **데이터명** | 전국 어린이집 정보 조회 API / 전국어린이집표준데이터 |
| **API 제공처** | 공공데이터포털 (data.go.kr) |

| **인증방식** | 공공데이터포털 회원가입 + API 활용신청 (1-2시간 승인) |
| **공간 해상도** | 시군구 단위, 위치 좌표 제공 |
| **갱신 주기** | 정기 갱신 (세부 주기는 데이터 페이지 확인 필요) |

**주요 필드**:
- 시도, 시군구, 어린이집명
- 시설구분 (국공립, 민간, 가정, 직장)
- 수용인원, 현원인원, 보육실 수
- 보육교직원 수 (보육교사, 원장 등)
- CCTV 설치 수
- 셔틀버스 운영 여부
- 주소, 전화, 팩스, 홈페이지
- 위치 좌표 (위도/경도)

**활용 지표**:
- 어린이집 밀도 (상권 내 500m 반경)
- 보육 수용능력 (배후 영유아 인구 대비)
- 시설 질 평가 (CCTV 설치율, 교직원 대비 아동 비율)

|------|------|
| **제공기관** | 국토교통부 |

| **무료/유료** | 무료 |

| **갱신 주기** | 월 1회 이상 (신고 자료 기반) |
| **공간 해상도** | 동(행정동) 단위 |

- 거래유형 (매매, 전세, 월세)

- 층수, 방개수

**활용 지표**:
- 평균 실거래가 추이 (반년 평가)
- 거래량 (거래 활발도 지표)
- 신축 vs 기존 건물 비율

**기술 참고**:
- 현재 프로젝트에서 이미 활용 중
- XML 파싱 필요 (기존 코드 참고)

## 📝 기술적 고려사항

### API 신청 프로세스 (공통)
1. 공공데이터포털(data.go.kr) 또는 해당 기관 포털 회원가입
2. API 활용신청 버튼 클릭
3. 개발/운영 구분 선택
4. 1-2시간 (일부 24시간) 내 승인
5. 인증키(API Key) 발급
6. 테스트 후 운영 전환

### 주의사항
- **트래픽 한도**: 많은 API가 1일 최대 호출 건수 제한 (통상 10,000~1,000,000)
- **캐싱 필수**: 동일 지역 반복 조회 시 Redis 캐싱 필수
- **갱신 주기**: 월 단위 / 분기 단위 / 연 단위 차이 확인
- **공간 해상도**: 동(행정동) vs 격자(격자) vs 개별(지번) 구분
- **포맷 변환**: XML, JSON, WMS/WFS 등 다양한 포맷 대응 필요

## 📌 결론

**신규 연동 가능한 공공데이터 API는 최소 15개 이상 발굴됨.**

현 프로젝트의 지표(활력도, 경쟁강도, 생존율, 배후인구, 인프라접근성)를 보완/강화하기 위해:

1. **배후세대(주거밀도 강화)** → 건축물대장 API
2. **정확한 폐업 현황(생존율 강화)** → 국세청 + SGIS 격자
3. **이동수단 다양성(활력도 확장)** → KT 수도권 생활이동 (수도권 한정)
4. **교육/의료/주차(라이프스타일 신지표)** → 어린이집, 학교, HIRA, 주차장 API

이 중 **건축물대장, SGIS, KT 데이터**는 높은 지표 개선 효과를 기대할 수 있으므로 우선 검토 권장.
