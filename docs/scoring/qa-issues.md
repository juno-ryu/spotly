## 발견된 이슈 (우선순위순)

### ISS-01: KOSIS 인구 시군구 fallback 빈번 (P1) — ✅ 수정 완료

**현상**: 10건 중 4건에서 시군구 fallback 발생
- 강남역: 556,330명 (강남구)
- 홍대: 357,616명 (마포구)
- 해운대: 372,604명 (해운대구)
- 신촌역: 300,147명 (마포구)

**정상 반환된 곳**: 노원역(18,580명), 수원역(9,777명), 제주(46,102명), 광주(13,446명), 춘천(8,836명), 대전(3,319명)

**패턴**: 서울 중심 상업지구/지하철역 좌표에서 fallback 빈번. 주거 밀집 지역은 정상.

**근본 원인 (구조적 버그 — API 일시 실패가 아님)**:

2중 버그로 확인됨:

1. **카카오 JS SDK `coord2RegionCode`가 특정 좌표에서 H(행정동) 타입을 반환하지 않음**
   - 지하 좌표 (강남역 "강남대로 지하 396", 신촌역 "신촌로 지하 90")
   - 도로 중앙/경계 좌표 (홍대 "양화로 156", 해운대 "중동2로 11")
   - 카카오 SDK가 이 좌표들에서 B(법정동)만 반환, H(행정동) 미반환
   - `useReverseGeocode.ts`: `hRegion?.code ?? null` → null 전달

2. **`actions.ts`가 서버 `coordToRegion` 결과의 `adminDongCode`를 무시**
   - `districtCode`가 있으면 `coordToRegion` 자체를 호출하지 않아 서버 보완 기회 없음

**수정 내용 (2026-03-15)**:
- `actions.ts`: `districtCode && adminDongCode` 둘 다 있을 때만 클라이언트 값 사용
- 하나라도 없으면 서버 `coordToRegion` 호출 → 서버 REST API가 H타입 반환율 높음
- `adminDongCode: region.adminDongCode ?? parsed.data.adminDongCode` (서버 우선)

**미해결**: 서버 REST API도 H 타입 미반환하는 극소 좌표가 있을 수 있음. 실 테스트로 확인 필요.

### ISS-05: 대전역 지하철 데이터 미반영 (P1) — ✅ 수정 완료

**현상**: 대전역 1호선 바로 앞(대전 동구 정동 1-1)인데 지하철 정보 없음. 버스만 표시.

**근본 원인 (구조적 문제 — 역명 매칭 버그)**:

`DAEJEON_STATION_MAP` 값과 매칭 로직의 불일치:
- `normalizeStationName`이 카카오 결과 `"대전역 1호선"` → `"대전"` 반환
- `getDaejeonSubwayTraffic`: `name === stationName.replace(/역$/, "")` 로 비교
- `DAEJEON_STATION_MAP` 1104번 항목: `"대전역"` (역 포함 저장)
- map 값에는 `replace` 미적용 → `"대전역" === "대전"` → **매칭 실패**

**수정 내용 (2026-03-15)**:
- `subway/client.ts`: 양쪽 모두 `replace(/역$/, "")` 적용하여 통일
- 변경: `name.replace(/역$/, "") === stationName.replace(/역$/, "")` → `"대전" === "대전"` 일치

### ISS-06: 부산 해운대에 "서강대학교" 표시 (P2) — ✅ 수정 완료

**현상**: 부산 해운대(중동2로 11)인데 "서강대학교 1511m" 표시.

**근본 원인 (데이터 품질 + 필터 부족)**:

카카오 Places에 서강대 부속시설(평생교육원, 기술창업원 등)이 해운대에 등록되어 있고, 현재 필터가 본교/분교/부속기관을 구분하지 못함.

- `adapter.ts`: `place_name.includes("대학교") && category_name.includes("대학교")` — 부속기관도 통과
- `normalizeUnivName`: `"서강대학교 해운대평생교육원"` → `"서강대학교"` 로 잘라버림

**수정 내용 (2026-03-15)**:
- `university/adapter.ts`: 부속기관 키워드 블랙리스트 필터 추가
- `SUBSIDIARY_KEYWORDS = ["평생교육원", "기술창업원", "어학원", "사이버", "원격"]`
- `place_name`에 키워드 포함 시 필터에서 제외
- 검증 시 "캠퍼스"는 실제 분교(건국대 글로컬캠퍼스 등)를 오필터하므로 블랙리스트에서 제외함

### ISS-07: 수원역 버스 3노선만 표시 (P2) — 🔜 박사님 검토 필요

**현상**: 수원역 환승센터(덕영대로 924)인데 "3개 노선"만 표시. 실제로는 수십 개.

**근본 원인 (구조적 문제 — 정류장 조회 제한 + 승강장 분산)**:

수원역 환승센터는 승강장마다 별도 정류장 ID로 등록("수원역환승센터(12승강장)" 등). 현재 코드가 정류장을 5개만 가져오고, 그 중 대표 1개의 노선만 집계.

- `adapter.ts` 55줄: `numOfRows: 5` — 정류장 5개 제한
- `adapter.ts` 78~84줄: `routeCount` 기준 대표 정류장 1개 선택 → 해당 승강장 노선만 집계
- `client.ts` 186줄: `numOfRows: 50` — 노선도 50개로 제한, 단일 페이지만 요청

**수정 방안**:
- `numOfRows`를 10~15로 확대 — 더 많은 승강장 포함
- 대표 정류장의 노선 수 대신, 반경 내 전체 정류장의 고유 노선 합산으로 변경
- `getSttnThrghRouteList` `numOfRows`를 100으로 확대 또는 페이지네이션 추가
- 노선 합산 변경은 스코어링에 영향 → 박사님 검토 대상
