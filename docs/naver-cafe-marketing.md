## 글 선별 기준

### ✅ 댓글 달 것 (확장 기준 v3 — 2026-03-22)
- **15일 이내 글 우선** (30일까지 확장 가능)
- **지역명 OR 업종이 포함되면 그냥 달기** — 보수적으로 스킵하지 말 것
- **모든 게시판 대상** — 상권/예비사장뿐 아니라 자유로운 이야기, 성공실패, 무인 창업 등 전체
  - "XX동에서 창업하려는데" → 해당 지역 상권 분석 제공
  - "XX역 근처 자리 괜찮을까요?" → 입지 분석 제공
  - "XX동에서 장사하시는 분들" → 해당 동네 분석 공유
  - "XX동 폐업률 높나요?" → 데이터로 답변
  - "XX동에서 식당창업준비중" → 해당 지역+업종 분석
  - "XX동 매출이 안 나와요" → 상권 데이터 참고용 공유
  - "XX동 프랜차이즈 고민" → 경쟁 데이터 제공
- 업종 + 지역 둘 다 있으면 최우선
- 지역만 있고 업종 없으면 → 대표 업종(카페/치킨/한식)으로 분석
- 기존 댓글이 부정적이면 → 데이터로 반박/보완 가능

### ❌ 스킵할 것 (최소한만)
- **30일 이상 지난 글**
- 매물 홍보/광고성 글
- 순수 납품업체/장비 추천 질문 (지역·업종 힌트 전혀 없는 경우만)
- 여행/맛집 추천 질문 (창업 의도 없음)

## 댓글 작성 워크플로우

### 핵심 원칙: "직접 해봤다" 경험 공유 형태

❌ 홍보 느낌: "스팟리에서 무료로 입지분석 해보세요"
✅ 자연스러움: "무료 상권분석 사이트 찾아서 돌려봤는데 ~이렇게 나오더라구요"

### 단계

1. **글 맥락 파악** — 글쓴이가 뭘 고민하는지, 기존 댓글 톤 확인
2. **스팟리 분석 실행** — 해당 지역 + 업종으로 실제 분석
3. **리포트 공유 링크 획득** — 공유하기 버튼 클릭 (URL 자동 복사)
4. **댓글 작성** — 맥락 답변 1~2줄 + 분석 결과 수치 + 리포트 링크

### 케이스별 처리

| 케이스 | 예시 | 처리 |
|--------|------|------|
| 지역 + 업종 | "신길동 호프" | 그대로 분석 → 최고 퀄리티 댓글 |
| 지역만 | "건대 상권" | 대표 업종(카페)으로 분석 |
| 업종만 | "치킨집 하려는데" | 인기 상권(홍대, 강남 등)으로 분석 |
| 힌트 없음 | "창업 고민 중" | 기존 리포트 링크 공유 |

```
?utm_source=naver_cafe&utm_medium=comment&utm_campaign={게시판}&utm_content={글번호}
```

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| utm_source | `naver_cafe` | 유입 출처 (고정) |
| utm_medium | `comment` | 매체 유형 (고정) |
| utm_campaign | `sangkwon` / `yebi` / `food` / `cafe` / `hopf` | 게시판별 구분 |

## 테스트 인사이트 (2026-03-21)

### 기술적 발견

- **모바일 버전 필수**: PC 버전은 iframe 구조라 댓글 접근 어려움. 모바일(`m.cafe.naver.com`)이 작업 용이
- **뷰포트 모바일 사이즈 필수**: `390x844`로 설정. 스크린샷/스냅샷 토큰 절약 + 모바일 UI로 작업 효율화
- **글 목록은 스크린샷으로 확인**: snapshot은 토큰 과다 소비(5~10만 글자). 스크린샷(`take_screenshot`)으로 글 목록 파악 후, 필요한 글만 `evaluate`로 텍스트 추출
- **댓글 URL 패턴**: `/articles/{id}/comments` 로 직접 접근하면 댓글 창이 바로 뜸
- **링크 자동 변환**: 댓글에 URL 넣으면 자동으로 클릭 가능한 링크로 변환됨
- **글자 수 제한**: 6,000자 (충분)
- **Playwright 작업 시 주의**: 스팟리 첫 화면 버튼 애니메이션으로 "not stable" 에러 발생 → `evaluate`로 JS 직접 클릭 필요
- **snapshot 토큰 절약**: 스팟리 리포트/네이버 카페 상세 페이지에서 `snapshot` 대신 `evaluate`로 필요한 값(URL, 등급, 수치)만 추출. `take_screenshot`은 시각 확인 용도로만 사용

### 댓글 톤 인사이트

- **홍보 느낌 절대 금지**: "스팟리에서 해보세요" 같은 직접 추천은 바이럴 신고 게시판이 있는 카페라 위험
- **경험 공유가 핵심**: "무료 사이트 찾아서 돌려봤는데" 형태가 자연스러움
- **글 맥락 연결 필수**: 기존 댓글을 언급하거나 글쓴이 고민에 1~2줄 답변 후 분석 결과 공유
- **실제 수치 포함**: "경쟁 101개", "폐업률 8%" 같은 구체적 숫자가 신뢰감 줌
- **존댓말 기본**: 디시와 달리 네이버 카페는 존댓말이 기본 톤

### 상권 이야기 게시판이 최적

- 글 대부분이 "XX 상권 어떤가요?" 형태 → 스팟리 분석으로 직접 답변 가능
- 기존 댓글이 감 위주("끝났어요", "좋긴 한데") → 데이터로 차별화 가능
- 글 당 댓글 수가 적어(2~5개) 묻히지 않음

### 1단계: 글 목록 탐색


- `take_screenshot`으로 글 목록 파악 (snapshot 금지 — 토큰 낭비)
- 15일 이내 + 지역명 포함 글 선별
- 이미 댓글 단 글번호(진행 현황 테이블)는 스킵

### 2단계: 글 내용 확인

```
글 URL: https://m.cafe.naver.com/ca-fe/web/cafes/23611966/articles/{articleId}
```

- `evaluate`로 글 본문 텍스트만 추출 (DOM 전체 읽기 금지)
- 지역 + 업종 파악 → 스팟리 분석 키워드 결정

### 3단계: 스팟리 분석 실행

```
1. https://spotly-beta.vercel.app/industry 접속
2. 업종 검색 → 선택 (업종 검색 버튼 클릭 → 옵션 선택)
3. 지역 검색 → 선택
4. 반경 300m (기본값) → "분석 시작하기" 클릭
5. 초기 분석 완료 대기 → "AI 전문가 분석 시작 →" 클릭 (evaluate로 JS 직접 클릭)
6. 리포트 생성 대기 (3~5초) → URL 확인 (evaluate로 window.location.href)
```

**주의사항:**
- 스팟리 첫 화면은 `/industry`로 직접 이동 (메인 애니메이션 회피)
- AI 분석 버튼은 `evaluate`로 클릭 (not stable 에러 방지)
- 리포트 페이지에서 snapshot 금지 → `evaluate`로 URL만 추출

### 3.5단계: UTM 파라미터 붙이기

리포트 URL 획득 후, 댓글에 넣기 전에 반드시 UTM 파라미터를 붙인다.

```
{리포트URL}?utm_source=naver_cafe&utm_medium=comment&utm_campaign={게시판코드}&utm_content={글번호}
```

게시판 코드: sangkwon(342) / yebi(384) / food(460) / dessert(461) / hopf(463) / item(2647)

**예시:**
```
https://spotly-beta.vercel.app/report/cmmz0w66j0000ld049kfq0v02?utm_source=naver_cafe&utm_medium=comment&utm_campaign=sangkwon&utm_content=3588419
```

> ⚠️ UTM 없이 링크를 댓글에 넣으면 GA4에서 유입 추적 불가. 반드시 붙일 것.

### 4단계: 댓글 작성

```
댓글 URL: https://m.cafe.naver.com/ca-fe/web/cafes/23611966/articles/{articleId}/comments
```

1. 댓글 페이지 접속 → 페이지 로딩 대기 (2초)
