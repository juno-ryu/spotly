주의: 발행 계획표에 ID가 있다는 이유만으로 완료 처리하지 않는다. 계획표 16번 이후 일부 ID는 아직 발행되지 않았을 수 있다.

### 미발행 큐 계산 스크립트

다음 명령으로 현재 남은 큐와 다음 배치를 계산한다.

```bash
node - <<'NODE'
require('dotenv').config();
const fs = require('fs');
const { PrismaClient } = require('@prisma/client');
const p = new PrismaClient();
const docs = fs.readFileSync('docs/naver-blog-marketing.md', 'utf8');

const done = new Set(
  docs.split('\n')
    .filter(line => line.includes('✅ 작성 완료') || line.includes('blog.naver.com/spotly/'))
    .flatMap(line => line.match(/[a-z0-9]{20,30}/g) || [])
);

function region(address) {
  const parts = String(address || '').split(/\s+/).filter(Boolean);
  const dong = parts.find(x => /동$|읍$|면$|가$/.test(x));
  if (dong) return dong;
  const d = parts.find(x => /구$|군$|시$/.test(x) && !/특별시|광역시/.test(x));
  return d || parts.slice(0, 2).join(' ');
}

function industryShort(name) {
  const n = String(name || '');
  for (const [needle, label] of [
    ['커피', '카페'],
    ['서양식', '양식'],
    ['일식', '일식'],
    ['한식', '한식'],
    ['분식', '분식'],
    ['중식', '중식'],
    ['치킨', '치킨'],
    ['피자', '피자'],
    ['기타음식', '음식점'],
    ['일반유흥', '주점'],
  ]) {
    if (n.includes(needle)) return label;
  }
  return n.replace('음식점', '').replace('전문점', '').replace('일반', '').trim() || n;
}

function campaign(regionName, industryName) {
  return `${regionName}_${industryName}`.toLowerCase().replace(/[^a-z0-9가-힣]+/g, '_');
}

(async () => {
  const all = await p.analysisReport.findMany({
    orderBy: { createdAt: 'desc' },
    select: {
      id: true,
      address: true,
      industryName: true,
      totalScore: true,
      aiReportJson: true,
      createdAt: true,
    },
  });

  const pending = all.filter(report => !done.has(report.id));
  console.error(JSON.stringify({
    total: all.length,
    done: done.size,
    pending: pending.length,
    nextId: pending[0]?.id,
  }, null, 2));

  const batch = pending.slice(0, 5).map(report => {
    const ai = report.aiReportJson || {};
    const rg = region(report.address);
    const ind = industryShort(report.industryName);
    return {
      id: report.id,
      address: report.address,
      region: rg,
      industryShort: ind,
      score: Math.round(report.totalScore || 0),
      verdict: ai.verdict || '',
      summary: String(ai.summary || '').slice(0, 180),
      direct: ai.competitorCount?.direct ?? null,
      indirect: ai.competitorCount?.indirect ?? null,
      franchise: ai.competitorCount?.franchise ?? null,
      revenue: ai.revenueEstimate?.monthlyPerStoreMaan ?? null,
      peak: ai.revenueEstimate?.peakTimeSlot || '',
      age: ai.revenueEstimate?.mainAgeGroup || '',
      closeRate: ai.survivalAnalysis?.closeRate ?? null,
      openRate: ai.survivalAnalysis?.openRate ?? null,
      risk: ai.riskWarnings?.[0]?.title || '',
      campaign: campaign(rg, ind),
    };
  });

  console.log(JSON.stringify(batch, null, 2));
})().finally(() => p.$disconnect());
NODE
```

### 네이버 글쓰기 자동화 핵심

- 현재 블로그는 `spotly`다.
- 글쓰기 URL: `https://blog.naver.com/spotly?Redirect=Write&`
- 에디터 iframe 찾기:
  - `page.frames().find(frame => frame.url().includes('PostWriteForm'))`
- 제목 입력 selector:
  - `.se-title-text`
- 본문 입력 selector:
  - `.se-module-text` 중 `글감과 함께` 텍스트가 있는 첫 요소
- 본문은 긴 텍스트라 `keyboard.insertText`보다 클립보드 붙여넣기가 안정적이다.
  - `navigator.clipboard.writeText(body)`
  - `Meta+V`
- 발행 버튼:
  - `frame.getByRole('button', { name: '발행' })`
- 최종 확인 버튼:
  - `button.confirm_btn__WEaBq`
- 발행 성공 확인:
  - URL이 `https://blog.naver.com/spotly/{postNo}` 형태로 바뀌어야 한다.
- 임시글 팝업이 뜨면 새 글 기준으로 `취소`를 누른다.

### 속도 제한 운영 규칙

2026-06-02에는 100건 발행 후 네이버 반복 등록 제한이 걸렸다. 다음부터는 속도를 낮춘다.

- 하루 권장 발행량: 20~40건
- 한 배치: 5건부터 시작, 안정적이면 10건까지
- 글 사이 대기: 30~90초 랜덤
- 배치 사이 대기: 5~15분
- 제한/캡차/비정상 팝업이 보이면 즉시 중단
- 실패 draft는 즉시 재시도하지 말고 실패 섹션에 남긴 뒤 다음 사이클로 넘긴다.

### 성공 기록 방식

발행 후 RSS와 현재 URL로 성공을 확인한 뒤 이 섹션의 표에 이어 붙인다.

```markdown
| 101 | {id} | {제목} | https://blog.naver.com/spotly/{postNo} |
```

실패한 글은 `중단/실패 처리`에 아래 형식으로 남긴다.

```markdown
- {id}: {제목} — {실패 사유}. 발행 URL 없음. 다음 작업 시 미발행 대상으로 다시 처리한다.
```

### 재개 순서

1. RSS 최신 글과 이 문서의 마지막 성공 URL을 비교한다.
2. DB 큐 계산 스크립트로 다음 미발행 ID를 확인한다.
3. 현재 글쓰기 화면에 draft가 있으면 발행하지 말고 필요 시 취소/정리한다.
4. `cmnpw7kee0000la04fs93c4z4`가 여전히 첫 큐인지 확인한다.
5. 제한이 풀렸는지 1건 테스트 발행한다.
6. 성공하면 5건 배치로 진행한다.
7. 각 배치 후 RSS/URL 확인 후 문서 표에 성공분만 기록한다.
8. 제한 조짐이 있으면 즉시 중단하고 실패 섹션을 갱신한다.
