## 2. 핵심 문제 진단

### 문제 1: YouTube 댓글 = 헛트래픽
- 90건 댓글 작성 → 127세션 유입 → **참여율 2.36%, 체류 0초**
- 127명 중 3명만 실제 참여. 나머지 124명은 아무것도 안 하고 이탈
- 원인: 유튜브 앱 내 웹뷰 자동 프리로드 / 봇성 트래픽 / 인앱 브라우저 즉시 닫힘
- **네이버 카페 1건 = YouTube 30건 가치** (참여율 71% vs 2.36%)
- 이전 GA4(3월 초)에서는 "유튜브 = 최고 효율 채널"로 판단했으나, **참여율 데이터를 보면 완전히 반대**

### 문제 2: 공유가 구조적으로 안 일어남
- 370명 중 **12명(3.24%)만 공유 시도**, 총 33회
- 리포트 본 119명 기준으로도 10%만 공유
- 카카오 공유 → 유입: 26건 (공유 1건당 0.79명 유입)
- **바이럴 계수 K ≈ 0.025** (1명이 0.025명 데려옴 = 사실상 0)
- 코드 확인 결과:
  - 공유 버튼: 우상단 FloatingActionGroup 내 40×40px 아이콘 (h-10 w-10)
  - **리포트 하단에 공유 CTA 전혀 없음** (report-viewer.tsx 확인)
  - 리포트를 다 읽은 시점 = 공유 의향 최고점인데, 그 시점에 유도가 제로

### 문제 3: 재방문 거의 없음
- 재방문 42명 / 371명 = **11.3%**
- 분석 이력 접근: 9명 (2.4%)
- 이메일/푸시/리마인더 없음
- 리포트 이후 다음 행동 유도 전무

### 문제 4: 퍼널은 건강, 출구에서 100% 새어나감
- 퍼널 74% 전환율은 훌륭
- 문제는 **리포트 이후**: 공유 3.24%, 재방문 11.3%, 재분석 거의 0


### 연관 업종 추천 로직

**핵심 관점**: 사용자는 지역·업종이 이미 확정된 상태. "다른 지역", "다른 업종"은 의미 없음.
하지만 **내 업종에 영향을 주는 주변 업종 현황**은 실질적 인사이트:
- 카페 → 음료, 베이커리 (직접 경쟁 + 시너지)
- 약국 → 병원, 한의원 (수요 연동)
- 학원 → 어학원, 예체능학원 (학군 시너지)
- 미용실 → 네일숍, 피부관리 (뷰티 클러스터)
- 치킨 → 피자, 분식 (배달 경쟁)
- 편의점 → 슈퍼마켓 (직접 경쟁)

**구현 방식**: AI 리포트 생성 시 함께 추천

현재 AI 리포트 생성 프롬프트에 한 줄 추가:
```
"이 업종과 연관된 주변 업종을 2~3개 추천해줘.
경쟁이 될 수 있거나 시너지가 될 수 있는 업종으로.
반드시 다음 목록에서만 선택: [카페, 치킨, 한식, 중식, 미용실, 양식, 편의점, 베이커리, 음료, 일식, 피자, 분식, 햄버거, 약국, 병원, 학원, ...]"
```

→ 지원하는 업종 이름만 넘기면 토큰 200개 이하
→ 별도 API 호출 없이 기존 리포트 응답에 필드 하나 추가
→ aiReportSchema에 `relatedIndustries` 필드 추가

**응답 스키마 추가안**:
```ts
relatedIndustries: z.array(z.object({
  name: z.string(),        // "음료"
  reason: z.string(),      // "카페와 직접 경쟁 — 음료 전문점 현황 확인 필요"
})).optional()
```

**CTA 문구 예시**:
- 카페 리포트 → "카페 주변에 음료·베이커리가 얼마나 있는지도 중요해요"
- 약국 리포트 → "근처 병원 수가 약국 매출에 직접 영향을 줘요"
- 학원 리포트 → "같은 학군 내 어학원 밀집도도 확인해보세요"

### 공유 메시지 개선

## 6. GA4 심층 분석 — "다크 소셜" 발견

### 직접 유입(direct) 203세션의 정체

GA4에 "방문 페이지 + 쿼리 문자열" 2차 원 추가 결과:

| 랜딩 페이지 | 세션 | 참여율 | 의미 |
|---|---|---|---|
| direct → `/` | 114 | 78.95% | 홈으로 직접 접속 (북마크/검색/카톡 공유) |
| direct → `/report/cmn30qxjz...` | 17 | 70.59% | **특정 리포트 URL 직접 접속 = 카톡 공유** |
| spotly/kakao → `/report/cmn4pwer...` | 16 | 65.38% | 공유 버튼 경유 (UTM 있음) |

**핵심 발견**: 17세션이 특정 리포트 URL로 직접 유입 = **사용자가 공유 버튼 대신 URL 복붙으로 카톡 공유 중**

→ 공유 버튼(UTM 있음)으로 유입된 26건 외에, **UTM 없이 복붙으로 공유된 트래픽이 direct에 숨어있음**
→ 리포트 URL 직접 접속은 타이핑 불가능한 주소 (cmn30qxjz...) → 100% 공유 유입

### 실제 바이럴 계수 재계산

| 측정 방식 | 수치 | 비고 |
|---|---|---|
| report_share 이벤트 기준 | K ≈ 0.025 | 공유 버튼만 추적 |
| direct → /report/* 포함 시 | K ≈ 0.07~0.10 | 다크 소셜 포함 추정 |

**→ 실제 공유는 측정치의 3~4배 발생 중. 문제는 공유가 안 되는 게 아니라 "추적이 안 되는 것"**

### 다크 소셜 추적 방안 (설계)

1. **서버사이드 리포트 조회 로깅**: 같은 report ID에 서로 다른 IP/세션이 짧은 시간 내 접속 → "공유됨"으로 태깅
2. **리포트 URL에 뷰어 파라미터 자동 추가**: 리포트 페이지 최초 로드 시 `?ref=shared` 같은 파라미터를 OG 미리보기 URL에 포함 → 카톡 미리보기 클릭 시 UTM 없어도 추적 가능
3. **숏링크 시스템**: `spotly.website/r/ABC123` → 리다이렉트 시 UTM 자동 부착. 공유 시 이 URL 사용 유도
4. **GA4 커스텀 이벤트**: /report/* 페이지에 referrer 없이 도착 시 `dark_social_view` 이벤트 발화

## 8. 90일 실행 로드맵

| 주차 | 액션 | 투입 시간 | 기대 효과 |
|---|---|---|---|
| 1~2주 | 리포트 하단 CTA + 공유 메시지 개선 + 다크소셜 추적 코드 | 개발 6~8시간 | 공유율 3%→10%, 추적 가능 |
| 1~4주 | 네이버 지식iN 답변 시작 (매일 2~3건) | 1시간/일 | 월 100+ 유입 (누적) |
| 2~8주 | 네이버 블로그 주 2회 발행 (현재 4편→20편+) | 4시간/주 | SEO 임계점 돌파, 월 50+ 유입 |
| 3~4주 | 연관 업종 추천 (AI 프롬프트 + 스키마) | 개발 3~4시간 | 재분석율 상승, 세션/유저 비율 증가 |
| 3~4주 | 리포트 이메일 발송 + "상권 변동 알림" 설계 | 개발 4~6시간 | 재방문 11%→25% |
| 5~8주 | 소상공인진흥공단/세무사 파트너십 메일 | 2시간/주 | 파트너 1건당 월 50~200 유입 |

| 8~12주 | KISED 등재 + 추가 파트너십 | 1시간/주 | 장기 지속 유입 |

- 현재: 370 유저, 488세션, K ≈ 0.025, 재방문 11.3%
- 목표: 800~1,200 유저, 1,500+ 세션, K ≈ 0.15~0.20, 재방문 25%+
